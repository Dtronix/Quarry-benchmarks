# Quarry Benchmarks

Storage repo for the [Quarry](https://github.com/Dtronix/Quarry) benchmark dashboard.

## Live Pages

- **[Dashboard](https://dtronix.github.io/Quarry-benchmarks/dev/bench/)** — run-over-run trends for every Quarry benchmark, with time and allocated memory on independent axes. Click any point to jump to the full report for that commit.
- **[Run History](https://dtronix.github.io/Quarry-benchmarks/dev/bench/runs/)** — list of individual BenchmarkDotNet reports, one per commit.

## About

This repository stores the benchmark results published by CI from the main [Quarry](https://github.com/Dtronix/Quarry) repository. Each merge to `master` that touches benchmark-relevant code triggers a fresh run; the resulting data, per-commit report, and front-end pages are committed here to the `gh-pages` branch.

Benchmarks compare Quarry against **Raw ADO.NET**, **Dapper**, **EF Core**, and **SqlKata** on an in-memory SQLite database using [BenchmarkDotNet](https://benchmarkdotnet.org/). See the [benchmark methodology](https://dtronix.github.io/Quarry/articles/benchmarks.html) for category descriptions and how to run the suite locally.

## Layout

| Path | Contents |
|---|---|
| `dev/bench/data.js` | Historical runs — time (ns) and allocated bytes for every `Quarry_*` method |
| `dev/bench/runs/*.html` | Full per-commit BenchmarkDotNet reports (all implementations, not just Quarry) |
| `dev/bench/runs/runs.json` | Manifest of available report files |
| `index.html`, `dev/bench/index.html`, `dev/bench/runs/index.html` | Landing, dashboard, and run-history UIs |

## Do not edit directly

Everything in this repo is regenerated by [`.github/workflows/benchmark.yml`](https://github.com/Dtronix/Quarry/blob/master/.github/workflows/benchmark.yml) in the main Quarry repo. Front-end sources live in [`scripts/benchmark-pages/`](https://github.com/Dtronix/Quarry/tree/master/scripts/benchmark-pages) — edit them there. Changes pushed directly to this repo will be overwritten on the next benchmark run.
